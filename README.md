# NeuroClip

Автоматический видеомонтаж хайлайтов Call of Duty: Warzone.

## Статус проекта

**Готово (Шаг 1):**
- Модуль 3 — Телеграм-репортер ошибок ИИ (полностью рабочий, реальные запросы к Bot API)
- Модуль 1 — базовый скелет интерфейса: неймплейт, загрузка видео, плеер предпросмотра
  с перемоткой, слайдеры чувствительности ИИ, кнопка отправки репорта

**Готово (Шаг 2a):**
- Модуль 2, часть OpenCV — детектор низкого HP (красная виньетка) и детектор
  килфида (появление новой строки), кнопка «🔍 Анализировать» в интерфейсе,
  панель найденных моментов с переходом по клику, утилита калибровки ROI

**Готово (Шаг 2b):**
- Детектор взрыва/яркой вспышки (скачок яркости + тёплый цвет, классический CV)
- Детектор прицела снайперской винтовки (тёмная виньетка + проверка кругом Хафа)
- Обёртка под YOLOv8 — готова к использованию, если найдёте/обучите модель
  под нужные классы (см. честное примечание в yolo_detector.py)
- Детектор прямого взгляда в камеру через MediaPipe FaceLandmarker — требует
  ручной установки файла модели (см. assets/models/README.md), без него
  анализ продолжает работать остальными детекторами

**Готово (Шаг 2c):**
- Детектор резких громких звуков (Librosa: onset-детекция + RMS-фильтр по
  громкости относительно фона записи) — работает всегда, без опциональных
  зависимостей, кроме системного ffmpeg
- Детектор смеха через Whisper — ОТКЛЮЧЁН по умолчанию (чекбокс в интерфейсе),
  на порядок медленнее остального анализа и не является надёжным (см. честное
  примечание в laughter_detector.py)

**В разработке (следующие шаги):**
- Сбор собственного датасета через `label_tool.py` → `train_classifier.py`
  (готово, ждёт вашей разметки)

**Готово (Шаг 4):**
- Модуль 4 — сборка монтажа (MoviePy) и генерация превью (rembg + текст)

## Сбор своего датасета для взрыва/прицела

Готовой чужой модели под эту задачу в открытом доступе нет (проверено —
либо реальные фото пожаров, не игровые VFX, либо Warzone-датасеты, но
размеченные под детекцию игроков для автоприцеливания, то есть под читерский
софт — такие я осознанно не использую и не привожу на них ссылки). Поэтому
собираем свой датасет из ваших же записей:

```
python label_tool.py путь/к/матчу1.mp4 путь/к/матчу2.mp4
```

Инструмент сам найдёт кадры-кандидаты (через уже работающие детекторы на
повышенной чувствительности — считай, "предположения", не факт) плюс немного
случайного фона для баланса классов, и откроет окно разметки: один кадр —
одно решение. Управление: `1` — взрыв, `2` — прицел, `3`/Delete — не
подходит, `Z`/Backspace — отменить последнее решение (ничего не удаляется
безвозвратно — "удалённые" кадры лежат в `dataset/_deleted/`, можно почистить
вручную когда убедитесь, что всё верно).

Можно закрывать окно в любой момент — неразмеченные кандидаты дождутся вас
при следующем запуске. Уже обработанные видео не сканируются повторно.

Когда наберётся достаточно примеров каждого класса (минимум 30 на класс,
для приличного качества лучше 150-300) — запустите:

```
python train_classifier.py
```

Скрипт сам проверит, хватает ли данных, сделает train/val разбиение и
дообучит YOLOv8-классификатор (transfer learning от ImageNet — быстрее и
качественнее, чем с нуля). В конце выведет путь к готовой модели и готовый
код для подключения в `modules/analysis_pipeline.py`.

## Установка

1. Установите Python 3.10 или новее: https://www.python.org/downloads/
   (при установке на Windows отметьте галочку "Add python.exe to PATH")

2. Установите ffmpeg (нужен для анализа звука — Модуль 2c):
   - Windows: скачайте с https://ffmpeg.org/download.html, распакуйте и
     добавьте папку `bin` в PATH
   - macOS: `brew install ffmpeg`
   - Linux: `sudo apt install ffmpeg`

3. Откройте терминал (Windows: PowerShell, macOS/Linux: Terminal) в папке `NeuroClip`.

4. Установите зависимости:

   ```
   pip install -r requirements.txt
   ```

5. Скопируйте `.env.example` в `.env` и заполните двумя значениями:

   - **TELEGRAM_BOT_TOKEN** — получите у `@BotFather` в Telegram: напишите ему
     `/newbot`, следуйте инструкциям, скопируйте выданный токен.
   - **TELEGRAM_CHAT_ID** — напишите новому боту любое сообщение, затем откройте
     в браузере `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates` и найдите
     число в поле `"chat":{"id": ...}`.

6. Запустите приложение:

   ```
   python main.py
   ```

## Важно про безопасность

Никогда не публикуйте файл `.env` с реальным токеном в открытом GitHub-репозитории —
любой, у кого есть токен, сможет отправлять сообщения от имени бота. Файл `.gitignore`
уже настроен так, чтобы исключить `.env` из публикации.

## Сборка монтажа и превью (Модуль 4)

После «🔍 Анализировать» становятся активны две кнопки:

- **🎬 Собрать монтаж** — вырезает отрезки вокруг каждого найденного момента
  (окно "до/после" разное для разных типов события — например, для килфида
  важнее показать подготовку к килу, для зрительного контакта — реакцию
  после) и склеивает через MoviePy. Отрезки, идущие близко друг к другу,
  автоматически объединяются в один непрерывный кусок вместо рваной нарезки.
- **🖼 Создать превью** — берёт кадр самого "зрелищного" момента (честный
  выбор среди уже найденных событий по весу типа × уверенности детектора,
  а не абстрактная метрика движения с нуля), убирает фон через rembg и
  накладывает автоматически сгенерированный заголовок в фирменном стиле.

Обе операции идут в фоновом потоке с прогрессом в статус-баре, программу
не подвешивают. При сохранении откроется стандартный диалог "Сохранить как".

**Честно про rembg:** при первом использовании скачивает модель сегментации
(~176 МБ). Если скачивание не удастся — превью всё равно соберётся, просто
с исходным фоном кадра вместо вырезанного объекта, программа не упадёт.

## Известные ограничения текущего шага (честно, без приукрашивания)

- **Предпросмотр без звука.** Плеер показывает кадры видео, но не воспроизводит аудио.
  Синхронизированное аудио потребует отдельного плеера (например, `python-vlc`) —
  скажите, если это нужно уже сейчас, а не после Модуля 2.
- **Точность перемотки** зависит от расстановки ключевых кадров в исходном видео —
  особенность работы со сжатым видео через OpenCV/FFmpeg, а не баг.
- **Слайдеры «Взрывы/прицел» и «Зрительный контакт»** пока сохраняют значения, но ни
  на что не влияют — соответствующие детекторы (YOLOv8, MediaPipe) появятся в Шаге 2b.
- **Координаты ROI килфида и толщина виньетки HP — отправная точка, не гарантия.**
  Точное положение HUD-элементов зависит от разрешения записи и настройки HUD Scale
  в игре, которые мне заранее не известны. Проверьте своей записью через
  `python calibrate_roi.py путь/к/видео.mp4` и поправьте координаты в
  `modules/detectors/killfeed_detector.py`, если рамки не совпадают.
- **OCR текста килфида — best-effort.** Требует отдельной установки Tesseract OCR
  (`apt install tesseract-ocr` на Linux, или инсталлятор UB-Mannheim на Windows —
  https://github.com/UB-Mannheim/tesseract/wiki). Без него сама детекция события
  «новая строка килфида» всё равно работает, просто без распознанного текста.
- **Детекторы взрыва и прицела — классический CV, не YOLO.** Предобученный
  YOLOv8 не различает "взрыв"/"прицел" (это не классы COCO), а обучить свою
  модель я не могу без размеченных кадров Warzone. Работают по яркости/цвету
  (взрыв) и геометрии тёмной виньетки с кругом Хафа (прицел) — проверены на
  синтетических кадрах с известным результатом, но это не нейросеть, точность
  ограничена тем, что видно по яркости/форме, а не пониманием сцены.
- **Детектор зрительного контакта — не протестирован end-to-end мной.**
  Математика оценки взгляда проверена отдельно (7/7 тестов на сфабрикованных
  координатах), но реальное распознавание лица MediaPipe требует файл модели
  (`assets/models/face_landmarker.task`), недоступный для скачивания из моей
  среды. Первый реальный прогон на живом лице — за вами.
- **Детектор звуковых пиков (Librosa) — полностью протестирован**, включая
  реальное извлечение звука через ffmpeg на видео с известными звуковыми
  событиями. Это единственный ML-компонент проекта без "домашнего задания"
  для вас — работает сразу.
- **Детектор смеха (Whisper) — честно ненадёжный по своей природе.** Whisper
  иногда помечает несловесные звуки как "[laughter]", но это не официальная
  функция, а побочный эффект обучающих данных — непоследовательно, чаще у
  крупных моделей. Отключён по умолчанию (чекбокс в интерфейсе) — не только
  из-за ненадёжности, но и потому что полная транскрипция на порядок медленнее
  остального анализа. Протестирована только логика поиска маркеров в тексте
  (18/18 на сфабрикованных строках) — саму транскрипцию проверить не смог
  (модель Whisper тоже не скачивается из моей среды — уже четвёртый ML-компонент
  подряд с этим ограничением, включая YOLO и MediaPipe).
- **Монтаж (MoviePy) — полностью протестирован**, включая реальный рендер и
  проверку итоговой длительности файла через ffprobe. Контекстные окна вокруг
  событий (сколько показать до/после) — разумные значения по умолчанию,
  не результат подбора на реальных Warzone-записях (у меня их нет) — можете
  захотеть поправить `_EVENT_WINDOWS` в `modules/montage_builder.py` под свой вкус.
- **Превью — выбор кадра и заголовок протестированы полностью** (чистая логика
  без внешних моделей). Удаление фона через rembg — та же история, что с
  YOLO/MediaPipe/Whisper: модель (~176 МБ) не скачивается из моей среды,
  пятый ML-компонент подряд с этим ограничением. У вас с обычным интернетом
  должно пройти штатно; если нет — превью всё равно соберётся, просто с
  исходным фоном кадра.
- Стиль "3D cinematic" реализован средствами 2D-интерфейса (неоновое свечение через
  настоящий Gaussian Blur, слои глубины, акцентные цвета) — полноценный 3D-рендеринг
  потребовал бы игрового движка/OpenGL, что не входит в десктоп-приложение на CustomTkinter.

## Как проверить, что детекторы реально работают

```
python calibrate_roi.py путь/к/вашему/видео.mp4 30
```

Откроет кадр на 30-й секунде и сохранит `roi_calibration_preview.png` с наложенными
зонами анализа — визуально проверьте, что жёлтая рамка (килфид) и красное кольцо
(виньетка HP) совпадают с реальными элементами HUD в вашей записи.

Затем в самом приложении: загрузите видео → «🔍 Анализировать» → найденные моменты
появятся в правой панели, клик по любому — перематывает плеер на этот момент.


## Сборка в .exe (Windows)

После первого запуска `start.bat` (нужно окружение с зависимостями) запустите:

```
build_exe.bat
```

Результат — папка `dist/NeuroClip/` (не один файл — так требует сама
CustomTkinter, см. комментарий в `neuroclip.spec`). Переносить/раздавать
нужно ВСЮ эту папку, не только .exe.

Честно: собрать и запустить exe я могу только на Linux (не Windows —
PyInstaller не кросс-компилирует), поэтому проверил структуру сборки и то,
что процесс запускается и не падает сразу, но настоящий Windows-прогон —
только на вашей стороне. Первая попытка может потребовать доводки.

```
├── dataset_extract.py              # извлечение кадров-кандидатов для разметки
├── label_tool.py                   # GUI разметки датасета (взрыв/прицел/фон)
├── train_classifier.py             # дообучение YOLO-классификатора на датасете
├── requirements.txt
├── .env.example                    # шаблон для секретов
├── modules/
│   ├── telegram_reporter.py        # МОДУЛЬ 3: отправка репортов в Telegram
│   ├── analysis_pipeline.py        # оркестратор: прогоняет детекторы по видео
│   ├── audio_extractor.py          # извлечение звука из видео через ffmpeg
│   ├── montage_builder.py          # МОДУЛЬ 4: склейка отрезков через MoviePy
│   ├── preview_generator.py        # МОДУЛЬ 4: превью (кадр + rembg + текст)
│   └── detectors/
│       ├── base.py                  # общий тип DetectionEvent
│       ├── hp_detector.py            # низкое HP (красная виньетка, HSV-анализ)
│       ├── killfeed_detector.py      # килфид (покадровый diff ROI + опц. OCR)
│       ├── explosion_detector.py     # яркая вспышка (скачок яркости + тёплый цвет)
│       ├── scope_detector.py         # прицел (тёмная виньетка + круг Хафа)
│       ├── yolo_detector.py          # обёртка YOLOv8 (для дообученной модели)
│       ├── gaze_geometry.py          # чистая математика оценки взгляда (без MediaPipe)
│       ├── eye_contact_detector.py   # интеграция с MediaPipe FaceLandmarker
│       ├── audio_peak_detector.py    # резкие громкие звуки (Librosa onset+RMS)
│       └── laughter_detector.py      # смех через Whisper (best-effort, опционально)
├── ui/
│   ├── theme.py                     # цвета и типографика
│   ├── nameplate.py                  # брендинг CASPER / Denis с неоновым свечением
│   ├── video_player.py               # плеер предпросмотра (OpenCV + Tkinter)
│   ├── analysis_panel.py             # список найденных моментов
│   ├── error_dialog.py               # окно "Ошибка ИИ"
│   └── main_window.py                # сборка главного окна
├── dataset/                         # НЕ в git — ваши размеченные скриншоты
│   ├── _staging/                     # кандидаты, ещё не разобранные
│   ├── _deleted/                     # отсеянные (корзина, не безвозвратно)
│   ├── explosion/ , scope/ , background/
└── assets/
    ├── fonts/
    │   ├── RussoOne-Regular.ttf      # шрифт неймплейта (лицензия SIL OFL 1.1)
    │   └── OFL.txt
    └── models/
        └── README.md                 # куда положить face_landmarker.task
```
